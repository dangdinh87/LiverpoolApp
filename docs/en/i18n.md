# Internationalization (i18n)

Bilingual support: **English (en)** and **Vietnamese (vi)** via `next-intl`. No URL prefix — `/squad` stays `/squad` in both languages.

---

## Configuration

**File:** `src/i18n/request.ts`

```typescript
export default getRequestConfig(async () => {
  const cookieStore = await cookies();
  const headerStore = await headers();

  // Detection order: cookie → Accept-Language header → fallback "en"
  const raw =
    cookieStore.get('NEXT_LOCALE')?.value ||
    detectLocaleFromHeader(headerStore.get('accept-language'));

  const locale = SUPPORTED_LOCALES.includes(raw) ? raw : 'en';

  return {
    locale,
    messages: (await import(`../messages/${locale}.json`)).default,
  };
});
```

**Detection order:**
1. `NEXT_LOCALE` cookie — set when user explicitly switches language (365-day expiry)
2. `Accept-Language` header — browser's preferred language (first supported match)
3. Fallback: `en`

**Supported locales:** `['en', 'vi']` — default `en`.

## Translation Files

| File | Purpose |
|---|---|
| `src/messages/en.json` | English strings |
| `src/messages/vi.json` | Vietnamese strings |

Both files share identical key structure. Top-level namespaces:

| Namespace | Contents |
|---|---|
| `Common` | Nav links, shared labels, error messages |
| `Hero` | Homepage hero section |
| `Bento` | Homepage bento grid widgets |
| `News` | All news page strings (feed, article, comments, translate, digest) |
| `Squad` | Squad page + position filter |
| `Fixtures` | Fixtures timeline + competition labels |
| `Standings` | League table |
| `Stats` | Stats dashboard labels |
| `History` | Club history page |
| `Profile` | User profile page |
| `Auth` | Login / register forms |
| `Gallery` | Gallery page |
| `Language` | Language switcher aria labels |

## Usage in Server Components

```typescript
import { getTranslations } from 'next-intl/server';

export default async function SquadPage() {
  const t = await getTranslations('Squad');
  return <h1>{t('title')}</h1>;
}
```

`getTranslations` is async — must be awaited. Locale is resolved automatically from the request config.

---

## Usage in Client Components

```typescript
'use client';
import { useTranslations } from 'next-intl';

export function NewsFilter() {
  const t = useTranslations('News.feed');
  return <button>{t('loadMore')}</button>;
}
```

`useTranslations` is synchronous. Messages are pre-loaded server-side and hydrated to the client — no additional fetch needed.

### Parameterized strings

```typescript
// en.json: "summary": "{total} articles — {en} international, {vi} Vietnamese"
t('summary', { total: 120, en: 80, vi: 40 })
// → "120 articles — 80 international, 40 Vietnamese"
```

### Nested namespaces

```typescript
const t = useTranslations('News');
t('feed.loadMore')    // → "Load More"
t('digest.badge')     // → "AI Summary"
t('comments.empty')   // → "No comments yet. Be the first!"
```

| English | Vietnamese |

| `src/data/club-info.json` | `src/data/club-info.vi.json` |
| `src/data/player-bios.vi.json` | (Vietnamese only, no EN equivalent) |

Loading pattern in server components:

```typescript
import { getLocale } from 'next-intl/server';

const locale = await getLocale();
const data = locale === 'vi'
  ? (await import('@/data/trophies.vi.json')).default
  : (await import('@/data/trophies.json')).default;
```

## Language Switcher

**File:** `src/components/layout/LanguageSwitcher.tsx` (client component)

Rendered inside `NavbarClient`. Displays current locale code (EN/VI) with a dropdown.

**Switching logic:**
```typescript
const switchLocale = (code: string) => {
  Cookies.set('NEXT_LOCALE', code, { expires: 365 });  // persist 1 year
  router.refresh();  // re-render with new locale (no page reload)
};
```

`router.refresh()` triggers a server re-render. `next-intl` picks up the new cookie and returns the correct translation file. The URL does not change.

## Adding New Translations

1. Add the key to `src/messages/en.json` under the appropriate namespace:
   ```json
   "MyFeature": {
     "title": "My Feature",
     "description": "Description here"
   }
   ```

2. Add the same key to `src/messages/vi.json` with the Vietnamese value:
   ```json
   "MyFeature": {
     "title": "Tính Năng Của Tôi",
     "description": "Mô tả ở đây"
   }
   ```

3. Use in a server component:
   ```typescript
   const t = await getTranslations('MyFeature');
   ```

   Or in a client component:
   ```typescript
   const t = useTranslations('MyFeature');
   ```

**Rules:**
- Both files must have identical key structure — missing keys in one locale will throw at runtime in development
- Use existing namespaces when possible; only create top-level namespaces for distinct page sections
- Parameterized values use `{paramName}` syntax
- Don't store HTML in translation strings — compose with JSX instead

## Cookie Persistence

| Cookie | Value | Expiry | Set by |
|---|---|---|---|
| `NEXT_LOCALE` | `"en"` or `"vi"` | 365 days | `LanguageSwitcher` via `js-cookie` |

The cookie is read server-side on every request via `cookies()` from `next/headers`. No client-side hydration mismatch because locale resolution happens entirely in the `getRequestConfig` function before rendering.
