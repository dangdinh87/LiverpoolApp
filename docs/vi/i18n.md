# Đa Ngôn Ngữ (i18n)

Hỗ trợ song ngữ: **Tiếng Anh (en)** và **Tiếng Việt (vi)** qua `next-intl`. Không có tiền tố URL — `/squad` vẫn là `/squad` ở cả hai ngôn ngữ.

---

## Cấu Hình

**File:** `src/i18n/request.ts`

```typescript
export default getRequestConfig(async () => {
  const cookieStore = await cookies();
  const headerStore = await headers();

  // Thứ tự phát hiện: cookie → header Accept-Language → fallback "en"
  const raw =
    cookieStore.get('NEXT_LOCALE')?.value ||
    detectLocaleFromHeader(headerStore.get('accept-language'));

  const locale = SUPPORTED_LOCALES.includes(raw) ? raw : 'en';

  return {
    locale,
    messages: (await import(`../messages/${locale}.json`)).default,
  };
});
```

**Thứ tự phát hiện ngôn ngữ:**
1. Cookie `NEXT_LOCALE` — được gán khi người dùng chủ động chuyển ngôn ngữ (hết hạn sau 365 ngày)
2. Header `Accept-Language` — ngôn ngữ ưu tiên của trình duyệt (khớp đầu tiên được hỗ trợ)
3. Fallback: `en`

**Các locale được hỗ trợ:** `['en', 'vi']` — mặc định `en`.

## File Bản Dịch

| File | Mục đích |
|---|---|
| `src/messages/en.json` | Chuỗi tiếng Anh |
| `src/messages/vi.json` | Chuỗi tiếng Việt |

Cả hai file có cấu trúc key giống hệt nhau. Các namespace cấp cao nhất:

| Namespace | Nội dung |
|---|---|
| `Common` | Link điều hướng, nhãn dùng chung, thông báo lỗi |
| `Hero` | Phần hero trang chủ |
| `Bento` | Các widget bento grid trang chủ |
| `News` | Tất cả chuỗi trang tin tức (feed, bài báo, bình luận, dịch thuật, digest) |
| `Squad` | Trang đội hình + bộ lọc theo vị trí |
| `Fixtures` | Timeline lịch thi đấu + nhãn tên giải |
| `Standings` | Bảng xếp hạng giải đấu |
| `Stats` | Nhãn trang thống kê |
| `History` | Trang lịch sử CLB |
| `Profile` | Trang hồ sơ người dùng |
| `Auth` | Form đăng nhập / đăng ký |
| `Gallery` | Trang thư viện ảnh |
| `Language` | Nhãn aria cho bộ chuyển đổi ngôn ngữ |

## Dùng Trong Server Components

```typescript
import { getTranslations } from 'next-intl/server';

export default async function SquadPage() {
  const t = await getTranslations('Squad');
  return <h1>{t('title')}</h1>;
}
```

`getTranslations` là hàm async — phải được `await`. Locale được tự động xác định từ cấu hình request.

---

## Dùng Trong Client Components

```typescript
'use client';
import { useTranslations } from 'next-intl';

export function NewsFilter() {
  const t = useTranslations('News.feed');
  return <button>{t('loadMore')}</button>;
}
```

`useTranslations` là đồng bộ. Các message được tải trước phía server và hydrate xuống client — không cần fetch thêm.

### Chuỗi có tham số

```typescript
// en.json: "summary": "{total} articles — {en} international, {vi} Vietnamese"
t('summary', { total: 120, en: 80, vi: 40 })
// → "120 articles — 80 international, 40 Vietnamese"
```

### Namespace lồng nhau

```typescript
const t = useTranslations('News');
t('feed.loadMore')    // → "Load More"
t('digest.badge')     // → "AI Summary"
t('comments.empty')   // → "No comments yet. Be the first!"
```

## Bản Dịch Dữ Liệu Tĩnh

Một số trang dùng file dữ liệu JSON có phiên bản theo ngôn ngữ:

| Tiếng Anh | Tiếng Việt |
|---|---|
| `src/data/trophies.json` | `src/data/trophies.vi.json` |
| `src/data/history.json` | `src/data/history.vi.json` |
| `src/data/legends.json` | `src/data/legends.vi.json` |
| `src/data/club-info.json` | `src/data/club-info.vi.json` |
| `src/data/player-bios.vi.json` | (chỉ có bản tiếng Việt, không có bản EN tương ứng) |

Mẫu tải trong server components:

```typescript
import { getLocale } from 'next-intl/server';

const locale = await getLocale();
const data = locale === 'vi'
  ? (await import('@/data/trophies.vi.json')).default
  : (await import('@/data/trophies.json')).default;
```

## Bộ Chuyển Đổi Ngôn Ngữ

**File:** `src/components/layout/LanguageSwitcher.tsx` (client component)

Được render bên trong `NavbarClient`. Hiển thị mã locale hiện tại (EN/VI) cùng dropdown.

**Logic chuyển đổi:**
```typescript
const switchLocale = (code: string) => {
  Cookies.set('NEXT_LOCALE', code, { expires: 365 });  // lưu 1 năm
  router.refresh();  // render lại với locale mới (không tải lại trang)
};
```

`router.refresh()` kích hoạt server re-render. `next-intl` đọc cookie mới và trả về file bản dịch đúng. URL không thay đổi.

## Thêm Bản Dịch Mới

1. Thêm key vào `src/messages/en.json` trong namespace phù hợp:
   ```json
   "MyFeature": {
     "title": "My Feature",
     "description": "Description here"
   }
   ```

2. Thêm key tương ứng vào `src/messages/vi.json` với giá trị tiếng Việt:
   ```json
   "MyFeature": {
     "title": "Tính Năng Của Tôi",
     "description": "Mô tả ở đây"
   }
   ```

3. Dùng trong server component:
   ```typescript
   const t = await getTranslations('MyFeature');
   ```

   Hoặc trong client component:
   ```typescript
   const t = useTranslations('MyFeature');
   ```

**Quy tắc:**
- Cả hai file phải có cấu trúc key giống hệt nhau — key bị thiếu ở một locale sẽ gây lỗi runtime trong môi trường development
- Ưu tiên dùng namespace hiện có; chỉ tạo namespace cấp cao nhất mới cho các trang/tính năng riêng biệt
- Giá trị có tham số dùng cú pháp `{paramName}`
- Không lưu HTML trong chuỗi bản dịch — dùng JSX để kết hợp thay thế

## Lưu Trữ Cookie

| Cookie | Giá trị | Hết hạn | Gán bởi |
|---|---|---|---|
| `NEXT_LOCALE` | `"en"` hoặc `"vi"` | 365 ngày | `LanguageSwitcher` qua `js-cookie` |

Cookie được đọc phía server trong mỗi request qua `cookies()` từ `next/headers`. Không xảy ra lỗi hydration phía client vì việc xác định locale diễn ra hoàn toàn trong hàm `getRequestConfig` trước khi render.
